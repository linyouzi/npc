# 具備情緒回饋與記憶紀錄的文字型 NPC 對話模擬系統

## 專題介紹

本專題旨在解決傳統 NPC 與現有 AI 對話系統常見的「失憶」與「情感單一」問題。系統核心結合大型語言模型（LLM）、階層式記憶轉化機制與人格導向情緒回饋設計，使 NPC 能夠在多次對話中累積長期記憶，並依據不同人格展現一致且自然的回應風格。

## 核心特色

### 🧠 階層式記憶轉化
- 當特定關鍵資訊在對話中被重複提及達**三次以上**，系統將自動轉化為長期記憶並保存
- 確保重要資訊不會遺失，建立持續的互動關係

### 😊 五種人格模式
- **溫柔**：溫和友善的對話風格
- **活潑**：充滿活力與熱情的回應
- **冷靜**：理性客觀的分析與建議
- **幽默**：風趣幽默的對話風格
- **嚴肅**：正式專業的溝通方式

### 💬 智能對話介面
- 流暢的聊天介面
- 整合人格設定、歷史記憶與即時輸入
- 產生自然且一致的對話回應

### 📊 記憶視覺化管理
- 查看、編輯或刪除 NPC 記憶
- 完整掌握對話歷史與記憶內容
- 提供透明的互動體驗

### 🔒 高度隱私保護
- 採用 **AES-256 加密**技術
- JSON 檔案儲存
- 確保資料安全與隱私保護

### ⚡ 輕量化部署
- 基於 Node.js 與 Express 架構
- 無需複雜資料庫
- 適合快速部署與擴展

## 系統架構

```
前端層
  ├─ 聊天介面
  ├─ 記憶視覺化管理
  └─ 人格切換控制
    ↓
後端層
  ├─ Node.js + Express
  ├─ Prompt 動態組裝器
  ├─ 對話 API 整合
  └─ 人格設定管理
    ↓
資料層
  ├─ JSON 檔案儲存
  ├─ AES-256 加密
  └─ 記憶管理機制
```

## 技術棧

- **前端**：HTML, CSS, JavaScript
- **後端**：Node.js, Express
- **AI 整合**：LLM API
- **資料儲存**：JSON 檔案
- **加密**：AES-256

## 應用場景

### 🎮 遊戲 NPC
為遊戲角色提供具有記憶與個性的對話系統，增強遊戲沉浸感與互動體驗。

### 📚 教育訓練
建立個性化教學助手，記錄學習進度與偏好，提供客製化的學習體驗。

### 🌐 語言學習
透過不同人格的對話練習，提升語言學習的趣味性與實用性。

### 🏫 校園互動
建立校園虛擬助手，協助學生與教職員進行日常互動與資訊查詢。

## 使用方式

1. 直接在瀏覽器中開啟 `index.html` 檔案
2. 或使用本地伺服器：
   ```bash
   # 使用 Python
   python -m http.server 8000
   
   # 使用 Node.js
   npx http-server
   ```
3. 在瀏覽器中訪問 `http://localhost:8000`
